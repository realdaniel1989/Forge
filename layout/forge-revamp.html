Design Revamp

Before → After

A screen-by-screen breakdown of every design decision that changed in this revamp — why it was made and what it improves for the user.

Global — All Screens

Before

256px sidebar with text labels on every page, consuming a third of screen width on most laptops.
All-caps everywhere — body copy, exercise names, button labels, nav items, card titles all uppercase.
font-weight: 900 used throughout, making every text element compete equally for attention.
Deeply nested box-in-box-in-box card structure adding visual noise without hierarchy benefit.
Green accent scattered across DONE badges, sync dots, borders, icon colours, link text simultaneously.
Hardcoded inline styles on every element, making the design inconsistent and hard to maintain.

After

60px icon rail — navigation collapses to icon-only, recovering ~196px of horizontal real estate per screen.
Uppercase only for metadata — set numbers, muscle tags, timestamps, and field labels. Names and sentences are mixed-case.
Weight scale: 400 / 600 / 700 / 800 — body text, labels, headings, and hero headings each occupy a distinct tier.
Single card-level per logical group — content uses size, opacity, and weight for hierarchy instead of nested boxes.
One red element per screen — the highest-priority action only. Everything else uses ghost or muted styling.
CSS custom properties (--variables) throughout — one token change propagates to every screen consistently.
01 Login

Before

Centered card floating in a black void with box-shadow — a modal disguised as a page.
Dumbbell icon logo inside a tiny bordered square — fragile and generic.
Single "Initialize Session" CTA — no email/password fields visible, implying SSO only.
All-caps subtitle "v2.4.0 High-Performance" placed as prominent text above the CTA.

After

Full-viewport layout — no floating card, left-aligned content with generous whitespace top and bottom.
Wordmark brand treatment — "Forge" bold weight 800, ".AI" light weight 400 — editorial and memorable.
Email + password fields with floating labels — standard login pattern users recognise immediately.
Contextual greeting "Good morning." — a warm human moment that replaces the version number as the emotional hook.
02 My Routines

Before

Grid of square cards — three equal-height boxy cards with no visual priority between them.
"Initiate Session" CTA identical on every card — no affordance for which routine to do next.
Exercise list in monospace with "»" bullets and "+3 MORE" as plain dim text — weak information hierarchy.
Navigation labels text repeated as large text in the sidebar consuming vertical space.

After

Horizontal list cards — compact rows, scannable at a glance. The most-recently-done routine expands to show preview exercises.
Red CTA on top card only — the next logical workout gets the primary action. Others get ghost buttons.
"Last done N days ago" metadata replaces the exercise count as the primary subtext — more useful at a glance.
Muscle group as pill tag (not prose uppercase) — colour-coded, compact, visually light.
03 Routines — Empty State

Before

Single dim text line in a bordered box — "No routines found. Generate one with AI or cascade custom structure." — no visual affordance.
No clear next action — the user must look to the sidebar navigation to figure out what to do.

After

Centred empty state with icon, descriptive copy, and two explicit CTA buttons: "Generate with AI" (primary) and "Build manually" (secondary).
Onboarding moment — the empty state teaches the two workflows available rather than leaving the user stranded.
04–05 AI Generator + Result

Before

Single text input — "Target Muscle Group" — as the only parameter. Too simple for the promise of AI generation.
Instruction subtext "Define target parameters. Generation engine will construct protocol." — verbose, jargon-heavy.
Exercise cards with full instruction paragraphs inside the card header area — information overload before the user starts.
Green accent heading on the AI Generator title — inconsistent with the page colour logic.

After

Two parameter tiles (Muscle Group + Experience Level) plus an optional Goal field — more structured and trustworthy AI input.
Heading is a human sentence: "Build a program." — conversational, confident, no jargon.
Exercise rows are scannable — number, name, scheme (4×8). Coaching notes visible below but not in the card title row.
Save Routine CTA in header — always visible without scrolling to the bottom of a long generated list.
06 Custom Routine Builder

Before

"Commit Sequence" label on save button — unnecessarily cryptic. "Abort" for cancel — too aggressive a term for a form.
Form inputs use JetBrains Mono inline in a card that's otherwise Syne — jarring typographic switch mid-layout.
White background CTA — breaks the dark aesthetic and introduces an inconsistent button style used nowhere else.

After

"Save Routine" and "Discard" — plain, unambiguous language. No jargon gatekeeping basic actions.
Field labels in Space Mono, values in Syne — consistent with the system-wide pattern: mono for metadata, sans for content.
Red full-width CTA at the bottom — the single action button system applied consistently to builder forms.
07 Live Workout

Before

All exercise cards equally weighted — no visual distinction between the active set and future exercises.
"01 / 4" and "DONE" badges as separate UI elements — set tracking was spread across the card header and table rows.
Green "DONE" badge and green "Finish Session" button — two greens with different meanings.
Timer input in the header buried among four other controls — hard to find during a set.

After

Active card full-opacity, upcoming cards dimmed to 45% — focus is always on the current exercise without hiding future context.
Set progress squares in the card header — a 4-square row where filled = done, empty = pending. Instantly readable.
Single red "Done" button per set row, "Finish Session" in the top bar. No colour collision.
Volume counter in the footer — passive data (total lbs lifted) lives at the bottom, not crowding the action area.
08 Rest Timer

Before

Floating corner widget — the timer appears as a small panel bottom-right. Readable only if standing directly over your phone.
Separate "low time" widget variant placed next to the normal one in the prototype — confusing duplication.
Exercise content still visible but dimmed behind the timer widget — split attention during a rest period.

After

Full-screen takeover — the timer is 80px, readable from 3 feet away mid-workout or on a phone propped on a rack.
Progress bar + "Next set" context — the athlete knows what's coming before the rest ends, no need to navigate away.
Low-time state is the same screen with the number and bar turning red — one component, two states, no layout change.
09 Analytics

Before

Green heading "Analytics Engine" with a trophy icon — tone-deaf for a utility data page.
All-caps day abbreviations and workout names inside calendar cells — dense and hard to scan.
Sets-volume in green on the weekly summary table — green used for data values creates a false "success" signal.

After

Plain heading "Analytics" with the date range as eyebrow text — functional, no performance theatre.
Mixed-case day names, large date numerals — easier to navigate the calendar grid, today highlighted with red accent.
Volume numbers in red — consistent with the single-accent-colour system. Red signals "the number you care about".
10–11 Modals (Analytics Detail + Add Exercise)

Before

4px border-radius on modals — they felt like dialog boxes, not layered sheets.
All-caps modal headers ("SELECT EXERCISE") at large letter-spacing — aggressive for an overlay.
Search input uppercase placeholder "SEARCH / ADD CUSTOM EXERCISE…" — shouting at the user.
No search icon on the Add Exercise input — the input purpose relies entirely on placeholder text.
Green accent-color on checkbox — the only remaining green after the Forge Crimson theme was applied, inconsistent.

After

12px border-radius on modals — the increased corner radius distinguishes them clearly as a floating layer above the page.
Sentence case modal headers ("Add Exercise") — calm, professional, consistent with the rest of the app.
Sentence case placeholder "Search exercises…" — natural language, not a command.
Search icon inside the input field — the input is visually self-describing without relying on placeholder text.
Red accent-color on checkbox — consistent with the single-accent system throughout the app.

Design principle throughout: Every change was driven by one question — does this help the user lift more, track better, or think less? Nothing was changed for aesthetic novelty alone. The crimson red accent, the tight icon rail, the full-screen rest timer, and the weight-scale typography all serve focus, not decoration.
